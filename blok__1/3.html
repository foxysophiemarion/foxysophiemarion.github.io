Архитектура «Чертогов Фрилансера» 4. Файлы и папки

Распаковав сборку, в корне вы обнаружите следующие файлы и папки:

1. Папка src — содержит исходные файлы и папки проекта. Именно здесь работает разработчик

2. Файл template.config.js — содержит пользовательские настройки ЧФ4

3. Файл README.txt — важная краткая информация по использованию ЧФ4

4. Файл cover.jpg — обложка ЧФ4

5. Файл package.json — содержит команды запуска сборщиков, информацию об установленных плагинах и их версиях, а также другую информацию об авторе и версии сборки. Не рекомендуется редактировать без соответствующих знаний

6. Файл docker-compose.yml — содержит инструкции по установке и запуску сервера в режиме fullstack для работы с CMS WordPress. Не рекомендуется редактировать без соответствующих знаний

7. Папка template_modules — содержит функционал сборки. Не рекомендуется редактировать

8. Файл vite.config.js — содержит инструкции для работы системы VITE. Не рекомендуется редактировать

9. При первом запуске любого режима появится папка .vscode, которая будет содержать настройки редактора и сниппеты (если это разрешено в настройках ЧФ4)

10. При первом запуске команды npm run zip появится папка zip, которая будет содержать zip-архив конечных файлов проекта

11. При первом запуске режима fullstack (npm run wp, npm run wpbuild) появится папка backend, которая содержит папки wordpress и database. Не рекомендуется редактировать без соответствующих знаний

12. После установки также автоматически появятся папка node_modules и файл package-lock.json. Не редактировать

Содержимое папки src:

1. Файл index.html — индексная (главная) страница проекта, который разрабатывается

2. Папка assets — используется для хранения медиафайлов проекта: изображений, шрифтов, иконок и т.д.

3. Папка styles — содержит общие файлы стилей проекта

4. Папка js — содержит общие файлы скриптов проекта, а также папки и файлы библиотек и фреймворков

5. Папка pug — содержит файлы для работы с препроцессором PUG

6. Папка php — содержит PHP-компоненты и является главной папкой PHP-сервера, если такое настройка включена

7. Папка files (пустая) — все файлы этой папки будут скопированы в папку с результатом (dist/files)

8. Папка components — содержит все HTML/CSS/SCSS/JS компоненты ЧФ4, а также тему для CMS WordPress, шаблоны HTML-страниц и их основной контент

Содержимое папки src/components:

Содержит все HTML/CSS/SCSS/JS компоненты ЧФ4, а также тему для CMS WordPress, шаблоны HTML-страниц и их основной контент

1. Папка custom (пустая) — здесь можно создавать собственные компоненты

2. Папка wordpress — содержит тему для CMS WordPress, плагин и вспомогательные файлы. Используется для разработки интерфейса под CMS WordPress в режиме fullstack (npm run wp, npm run wpbuild)

2.1 fls-theme — тема для CMS WordPress

2.2 fls-plugin — собственный плагин для CMS WordPress

2.3 JS-файлы — вспомогательные файлы, не редактировать, не удалять

3. Папка templates — содержит шаблоны HTML-страниц, можно добавлять свои

3.1 Папка main — содержит HTML/SCSS/JS файлы основного HTML-шаблона проекта

3.2 Папка projectpage — содержит HTML-шаблон страницы проекта

4. Папка pages — содержит HTML/SCSS/JS файлы основного контента страниц проекта

4.1 Папка home — содержит HTML/SCSS/JS файлы основного контента индексной (главной) страницы проекта

5. Папка effects — содержит папки и HTML/SCSS/JS файлы компонентов, которые добавляют в проект различные эффекты. Можно добавлять свои

6. Папка forms — содержит папки и HTML/SCSS/JS файлы компонентов, которые добавляют в проект различный функционал для работы с формами и их элементами. Можно добавлять свои

7. Папка layout — содержит папки и HTML/SCSS/JS файлы компонентов, которые добавляют в проект различные части интерфейса. Можно добавлять свои

Дополнительно, папки компонентов могут содержать папки и файлы плагинов конкретного компонента, например src/components/layout/header/plugins/scroll

Содержимое папки src/assets:

1. Папка fonts (пустая) — используется для конвертации и добавления локальных шрифтов в проект.

2. П апка img (пустая) — используется для хранения и использования изображений

3. Папка svgicons (пустая) — используется для конвертации и добавления в проект иконного шрифта и/или спрайта

Содержимое папки src/styles:

Файл style.scss — главный файл стилей, который подключается ко всем HTML-страницам.

Файл settings.scss — файл настроек общих значений стилей проекта

Папка fonts — содержит автоматически заполняемые, при запуске любого режима, CSS-файлы стилей подключения локальных и иконного шрифтов

Папка includes — содержит общие CSS/SCSS-файлы стилей

1. Файл index.scss — автоматически подключается к каждому SCSS-файлу проекта и содержит подключение настроек, SCSS миксинов и шаблонов. Можно подключать свои

2. Файл extends.scss — содержит набор SCSS-шаблонов. Можно добавлять свои

3. Файл mixins.scss — содержит набор SCSS-миксинов. Можно добавлять свои

4. Файл spritemap.scss — содержит SCSS-миксин для работы с SVG-спрайтом. Заполняется автоматически

1. Папка libs — содержит автоматически подключаемые к HTML-страницам CSS-файлы

1.1 Файл reset.css — автоматически подключается к каждой HTML-странице, если не используется другая CSS-библиотека/фреймворк

1.2 Файл tailwind.css — автоматически подключается к каждой HTML-странице, если использование указано в настройках ЧФ4

Содержимое папки src/js:

1. Файл app.js (пустой) — основной файл общих для всего проекта скриптов. Подключается к каждой HTML-странице

2. Папка common — содержит файлы общих скриптов

2.1 Файл functions.js — содержит полезные функции, которые используются в компонентах ЧФ4

3. Папка react — содержит стартовые файлы и папки для работы с React

4. Папка vue — содержит стартовые файлы и папки для работы с Vue
